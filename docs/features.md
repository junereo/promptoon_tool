# 기능 명세서 (Features)

Promptoon Authoring MVP 앱은 주요 기능 단위(Feature-Sliced Design)로 웹 애플리케이션을 모듈화하고 있습니다.
주요 기능들은 웹 앱(`apps/web/src/features`)과 API(`apps/api/src/modules`)를 포괄하여 아래의 주요 맥락으로 구성됩니다.

## 1. 프로젝트 및 에피소드 관리 (Project & Episode)

- **프로젝트 목록 조회 (`projectList`)**: 사용자가 생성한 인터랙티브 웹툰/스토리(프로젝트) 목록을 조회합니다.
- **프로젝트 생성**: 새로운 스토리라인 및 타이틀을 입력하여 프로젝트를 초기화합니다.
- **에피소드 관리**: 하나의 프로젝트 안에서 여러 개의 에피소드를 분리하여 생성하고, 개별 편집합니다.

## 2. 에피소드 에디터 (Editor)

에피소드 내부의 내용(컷 및 선택지)을 캔버스 기반으로 편집하는 가장 핵심적인 기능입니다.

- **무한 캔버스 (Infinite Canvas)**: 시각적인 플로우 차트 형태로 스토리를 집필합니다.
  - **Cut (노드)**: 텍스트 내용, 에셋 이미지 URL 등 개별 장면(Scene). (`scene`, `choice`, `ending`, `transition`)
  - **Choice (엣지)**: 컷과 컷을 이어주는 연결 및 사용자 선택 스크립트.
- **낙관적 업데이트 (Optimistic Updates)**: 사용자가 노드의 위치를 이동하거나, 텍스트를 편집할 때 딜레이 없이 즉각적으로 UI가 반응하며, 백그라운드에서 자동 저장(Auto Save)됩니다.
- **정렬 변경 (Reorder)**: 에피소드 내의 컷 순서를 일괄적으로 변경하고 이 순서는 배치(Batch) API로 한 번에 처리됩니다. 

## 3. 배포 및 유효성 검사 (Publish & Validate)

- **유효성 검증 (Validation)**: 스토리를 배포(Publish)하기 전에 발생할 수 있는 문제를 사전에 검지합니다.
  - 시작 컷 없음 (`missing_start_cut`)
  - 시작 컷 중복 (`multiple_start_cuts`)
  - 도달할 수 없는 컷 (`unreachable_cut`)
  - 막다른 길(엔딩이 아닌데 끝나는 경로) (`dead_path`) 등
- **배포 (Publishing)**: 유효성 검증을 통과한 에피소드는 최종 조회용 매니페스트(JSON 스냅샷 형태)로 고정되어 발행됩니다. (Draft 상태의 스냅샷)

## 4. 뷰어 (Viewer)

배포된 에피소드를 사용자가 실제로 플레이하고 읽는 인터페이스입니다.

- 스토리는 화면을 탭하거나, **선택지(Choice)** 버튼을 눌러 다음 노드로 분기하며 전개됩니다.
- 텔레메트리 (이벤트 수집): 사용자가 어떤 장면을 읽었는지(`cut_view`), 어떤 선택지를 골랐는지(`choice_click`), 무슨 엔딩을 보았는지(`ending_reach`) 익명으로 이벤트를 수집합니다.

## 5. 분석 (Analytics)

- 뷰어에서 수집한 텔레메트리 이벤트를 바탕으로 작성자(통계 조회자)에게 대시보드를 제공합니다.
- **Funnel 분석**: 시작 뷰어, 선택 뷰어, 엔딩 도달 뷰어를 통해 완독률(`completionRate`)을 제공.
- **일별 뷰어 조회 수 (`dailyViews`)**: 트래픽 확인 기능.
- **선택지 통계 (`choiceStats`)**: 특정 컷에서 어떤 선택지가 가장 많이 눌렸는지 비율(%) 집계 제공.
