📌 두 종류의 CTA 정리

1. 인앱 버튼 (앱 사용자 → 저장 액션)
• 위치: 화면 하단 고정
• 카피: "⬇ 저장하고 공유하기"
• 역할: 결과를 갤러리에 저장 → 사용자가 SNS에 직접 업로드

2. 카드 안 INFLOW BOX (SNS viewer → promtoon.ai 유입)
• 위치: 카드 하단의 점선 박스 (호텔 체크인 카운터 느낌)
• 컨텐츠: CHECK IN | promtoon.ai | "반응형 웹툰"
• 역할: SNS에서 본 사람이 어디로 가야 하는지 명확히 표시

유입 메커니즘 (질문 답변)
Viewer가 promtoon.ai로 가는 방법:
  • ① 카드 이미지 안의 URL을 보고 외움/타이핑 ← 핵심. 도메인이 짧아서 가능
  • ② 공유 텍스트에 URL 자동 포함 ← Web Share API 도입 시 클릭 가능
  • ③ 해시태그 #promtoon ← 검색 유입
→ ①번이 가장 확실. 그래서 카드 안 INFLOW BOX가 시각적으로 부각되어야 함.

왜 "CHECK IN" 컨셉인가
• 호텔 리플레이스의 세계관과 일치 ("체크인/체크아웃")
• 점선 보더 = 영수증/탑승권 느낌 → 현실적 디테일
• "반응형 웹툰" = 다음 작품도 있다는 암시 (시리즈성)

변경된 부분 (이전 안과 비교)
• ❌ "기록자 - 김작가" 제거 → ✅ "by 김작가" 작은 byline으로 이동
• ❌ "▶ 호텔에 체크인하기" CTA 박스 제거 (인앱 버튼과 카피 충돌)
• ✅ INFLOW BOX 신설 (viewer를 위한 유입 안내)
• ✅ promtoon.com → promtoon.ai (정정)
• ✅ 도메인이 짧아서 외우기 쉬움 → 큰 폰트로 부각
